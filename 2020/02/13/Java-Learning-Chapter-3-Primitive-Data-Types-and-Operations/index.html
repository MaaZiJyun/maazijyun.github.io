Structuring a Program

Develop efficient computer solutions to solve problems:

  1. Use Modules

  2. Use four logic structures

    a. Sequential structure

    ​ Executes instructions one after another in a sequence.

    b. Decision structure

    ​ Branches to execute one of two possible sets of instructions.

    c. Loop structure

    ​ Executes a set of instructions many times.

    d. Case structure

    ​ Executes one set of instructions out of several sets.

  3. Eliminate rewriting of identical process by using modules.

  4. Use techniques to improve readability including four logic structures, proper naming of variables, internal documentation and proper indentation.

Programming Style and Documentation

1. Appropriate Comments

a. Include a summary at the beginning of the program to explain what the program does, its key features, its supporting data structures, and any unique techniques it uses.

b. Include your name, class section, instructor, date, and a brief description at the beginning of the program.

2. Naming Conventions

a. Choose meaningful and descriptive names.
b. Variables and method names:

–Use lowercase. If the name consists of several words, concatenate all in one, use lowercase for the first word, and capitalize the first letter of each subsequent word in the name. For example, the variables radius and area, and the method computeArea.

3. Proper Indentation and Spacing Lines

a. Class names:

​ –Capitalize the first letter of each word in the name. For example, the class name ComputeArea.

b. Constants:

​ –Capitalize all letters in constants, and use underscores to connect words. For example, the constant PI and MAX_VALUE

c. Indentation

​ -Indent two spaces.

d. Spacing

​ -Use blank line to separate segments of the code.

4. Block Styles

Use end-of-line style for braces.

Identifiers

Special symbols to name such programming entities as variables, constants, methods, classes and packages.

Examples in the previous simple program:

  1. An identifier is a sequence of characters that consist of letters, digits, underscores (_), and dollar signs ($). _

  2. An identifier must begin with a letter, an underscore (_), or a dollar sign ($).

  3. An identifier cannot:

    ​ be a reserved word, e.g. true, false, or null

    ​ start with a digit

    ​ contain a (blank) space

    ​ contain special characters, except _ and $

  4. An identifier can be of any length.

  5. Conventions for choosing identifier names:

    ​ a. Name according to what it represents

    ​ b. Consistent in the use of variable names

    ​ c. Consistent when using upper and lowercase characters

    ​ –Variable name – lowercase, or mix-case

    ​ –Constant name – uppercase (all caps)
