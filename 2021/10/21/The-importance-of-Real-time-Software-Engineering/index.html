The importance of Real-time Software Engineering
2021-10-21

This is the first class of Real-time Software Engineering.

Basic Concept

Real-time Data

Real-time data (RTD) is information that is delivered immediately after collection. There is no delay in the timeliness of the information provided.

Example: Sensors: temperature, pressure, etc.

Real-time System (CMU)

RT systems span several domains of computer science.

  • defence and space systems, networked multimedia systems, embedded automotive electronics etc.

In a RT system the correctness of the system behaviour depends

  • not only the logical results of the computations,
  • but also on the physical instant (time) at which these results are produced.

A real-time system changes its state as a function of physical time, e.g., a chemical reaction continues to change its state even after its controlling computer system has stopped.

A real-time computer system must react to stimuli from the controlled object (or the operator) within time intervals dictated by its environment. The instant at which a result is produced is called a deadline
