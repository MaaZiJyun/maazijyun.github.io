User Interface and User Experience Week 3

2021-04-11

This week we missed one of our class but we finished a activity which also be presented on second class in this week.

For the activity, there are 2 sections.

Section A

  • Use examples from the previous class (system/app that gives positive UX and negative UX).
  • Assess the app using cognitive aspects (attention, perception, memory) lectured in class.
  • Copy the design implications item from attention, perception, memory into the given table (done for Positive UX app, please create the same table for Negative UX Apps).
  • Assess the app UI’s main feature based on the following instructions.

Describe in either column Clear or Vague/unclear representation in Table 1(Positive UX Apps) and 2 (Negative UX Apps). Screenshot the UI that shows the clear and vague indication on the UI in the appropriate column. If the UI shows vague representation, suggest ways to improve that particular context’s cognitive aspects with sketching at the Suggestion to improve column. Also, add more design implication into each cognitive aspect lacking from the given list in the table (if any).

Section B

Describe the mental model of groceries list to buy. List out each step you take to accomplish the goal.

Experience:
Regarding when people go for shopping in mental experience, people get in the shop to buy the things they in their plan or to just look around and to buy something randomly. They are shopping with a container, their hands, baskets or carts, to keep the item they choose to buy. they are walking around the shop to find the item which they want to buy, and the items will be put in container once they find it after they check the information. At the end of shopping, before they leave, they have to pay the money according to the items.

Then we put it into mental model:
Step 1. Browse and search items
Step 2. Find out more information of the items
Step 3. Add/Remove items to/from the shopping cart
Step 4. Checkout
Step 5. Choose how to pay and a shipping/delivery option
Step 6. submit order
Step 7. Get some type of feedback confirming the order was done
Step 8. The shopping process is done

Metaphor

In user interface design, an interface metaphor is a set of user interface visuals, actions, behaviours and procedures that exploit specific knowledge that users already have of other areas, in order to help users to interact with the user interface.

Verbal metaphor

Knowledge about a familiar domain in terms of elements and their relation to each other is mapped on to those in the unfamiliar one

Virtual metaphor

To create electronic counterparts to the physical objects in an office.

Verbal metaphor

Combine two metaphors or more together for carrying out a range of computer based actions
